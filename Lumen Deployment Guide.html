1

Download your project files

~1 min
↓

Download the project as a ZIP

Click the button below to download all of Lumen's files as a single ZIP archive. This includes everything: the app, the imaging engine, the new data layer, icons, and the PWA files.

Download Lumen.zip

What's inside

index.html, styles.css, manifest.json, sw.js, js/ (engine + core + UI), icons/, screenshots/. All photo processing code — nothing else. No server, no database.

2

Unzip the folder

Double-click Lumen.zip to unzip it. You should see a folder called lumen (or similar). Keep it somewhere easy to find — like your Desktop or Documents. Don't rename the files inside.

2

Create a GitHub repository

~5 min

GitHub is where your code lives. It's free, backs everything up automatically, and Netlify will deploy directly from it. Every time you save an update to GitHub, Netlify re-deploys automatically.

1

Create a free GitHub account (skip if you have one)

Go to github.com → click Sign up → choose the Free plan.

2

Create a new repository

Once signed in, click the + button in the top-right corner → New repository.

Fill in the form
→

Repository name: lumen (or lumen-travel-studio)

→

Visibility: Private (recommended — it's your personal app)

→

Initialize with README: leave unchecked

→

Click Create repository

You'll see an empty repo page

GitHub will show you a page with instructions. You can ignore those — the next phase covers uploading your files without using the command line.

3

Upload your files to GitHub

~5 min

Two options. Option A is the easiest (no install needed). Option B is better for ongoing development — you'll use it to push updates without re-uploading everything each time.

OPTION A Upload via GitHub website — simplest, no install
1

Open your new repo on GitHub

Go to github.com/YOUR-USERNAME/lumen. You'll see an empty repo with a message like "Quick setup."

2

Drag your files in

Click uploading an existing file (small link near the bottom of the page). Then drag the entire contents of your lumen folder into the upload area — not the folder itself, but all the files and subfolders inside it.

Include subfolders

Make sure js/, icons/, and screenshots/ are included. GitHub's web uploader supports folders — drag the whole project folder contents in one go.

3

Commit the files

Scroll down to Commit changes. Leave the default message or type Initial commit — Lumen v1. Click the green Commit changes button.

Your files are now on GitHub. You'll see them listed in the repo.

OPTION B GitHub Desktop — best for pushing future updates
1

Install GitHub Desktop

Download from desktop.github.com. It's a free app with a visual interface — no command line at all. Sign in with your GitHub account.

2

Clone your empty repository

In GitHub Desktop: File → Clone repository. Find lumen in the list. Choose a local path (e.g. your Documents folder). Click Clone.

3

Copy your Lumen files into the cloned folder

Open Finder/Explorer. Navigate to the cloned lumen/ folder. Copy all the files from your downloaded Lumen project into it.

4

Commit and push

Back in GitHub Desktop, you'll see all files listed as "Changes." Type a summary like Initial commit → click Commit to main → then Push origin. Done — files are on GitHub.

Future updates are easy

When Claude makes changes to your app, download the new files, copy them into the repo folder, commit in GitHub Desktop, and push. Netlify redeploys automatically within 30 seconds.

4

Deploy to Netlify

~5 min

Netlify hosts your app for free on a global CDN with HTTPS — both required for the PWA to work. It watches your GitHub repo and auto-deploys every time you push a change.

1

Create a free Netlify account

Go to netlify.com → Sign up → choose Sign up with GitHub (easiest — uses your existing account).

2

Add a new site from GitHub

In the Netlify dashboard: click Add new site → Import an existing project → Deploy with GitHub.

Authorize Netlify to access GitHub. Find and select your lumen repository.

3

Configure the build settings

Lumen is a static app — no build step needed. Leave all build settings blank:

→

Branch: main

→

Base directory: (leave empty)

→

Build command: (leave empty)

→

Publish directory: (leave empty, defaults to root)

Click Deploy lumen.

4

Your app is live

Netlify deploys in about 30 seconds. You'll get a URL like random-name-123.netlify.app. Click it — Lumen is live on the internet.

(Optional) Set a custom domain

In your Netlify site settings → Domain management → Add custom domain. You can use something like lumen.yourname.com if you own a domain. Netlify handles the HTTPS certificate automatically.

5

Install Lumen on every device

~2 min per device

Lumen is a PWA — it can be installed to the home screen or dock of every device and behaves like a native app: full-screen, offline-capable, no browser chrome.

iPhone / iPad

Open your Lumen URL in Safari. Tap the Share icon → Add to Home Screen. Tap Add. Lumen appears on your home screen like a native app.

Android

Open in Chrome. Tap the three-dot menu → Add to Home screen. Or Chrome may show an install banner automatically. Tap Install.

Mac / Windows

Open in Chrome or Edge. Look for the install icon in the address bar (a computer with a + sign). Click it → Install. Lumen opens in its own window.

Works offline after first visit

Once installed, Lumen launches instantly even without internet. The service worker caches the entire app shell on first load. Your photos and edits are always on-device.

→

Pushing future updates

30 seconds per update

When Claude makes changes to Lumen, here's how to push them live:

1

Download the updated file(s)

Claude will tell you exactly which files changed. Download only those.

2

Copy them into your repo folder

Replace the old files. Don't rename or move anything.

3

Commit and push in GitHub Desktop

Write a short message like Add album management → Commit → Push origin.

4

Netlify redeploys automatically

Within 30–60 seconds your live URL is updated. Reload Lumen on any device to get the update.

Next step: wire persistence (Phase 1)

The data layer is already written. The next build session will connect it to app.js so edits survive page refresh and photos persist between sessions. That's the last piece before the app is fully usable as a daily driver.
